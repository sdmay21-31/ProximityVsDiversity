User Guide

Astronomy researchers have difficulty visualizing the large heterogeneous datasets of binary stellar trajectories. With the vast number of publicly available datasets retrieved from astronomical observations and simulation, there needs to be a tool that allows researchers to quickly search for patterns in the data. The main use case for this data is the clustering of binary stars based on the proximity and diversity of the stellar attributes across multiple distance functions. Using these distance functions, analysts would like to measure the outcome of different attributes related to the attribute’s proximity vs. diversity. To solve this problem, we have developed an intuitive web-based tool that allows users to select distance functions for different data parameters. This application enables users to query a large heterogeneous dataset with a focus on proximity vs. diversity. In all, this website allows users to select a database, select 3 specific attributes within that dataset in relation to proximity and diversity, attach a weight to each attribute, select a time that the algorithm will cluster at, and select the number of clusters they are looking for. Once this is complete, a graph will be generated that represents the clustering of the attributes selected.

Proximity and diversity, in the context of this project, are how close and far two or more things are to each other. If you select mass in the proximity attributes and rad in the diversity attributes then you are looking for a clustering that is close in mass but far apart in radius.

You can select a database by clicking on the datasets under the blue dropdown at the top of the page labeled Select a Dataset. Click on the dataset you would like to use and you will be brought to the process page. You may switch databases at any time.

You can select attributes from the two tables labeled proximity attributes and diversity attributes by clicking on the check box next to the attribute name.

You can attach a weight to an attribute by typing a whole number into the “weight” section of the table corresponding to the attribute that is selected. Weight in this case will dictate the importance of an attribute (higher weight equals higher importance). The weight attribute will scale based on the other weights you have selected. In the case below Kstar has a weight of 1 however it will have an actual weight of 0.167, since Mass0 and Lumin have a weight of 2 and 3. The actual weight of Mass0 and Lumin are 0.333 and 0.5 respectively.

You can select the time you would like the attributes to be clustered at by typing a whole number between 0 and 3000 in the “time” box that is directly below the attribute tables.

You can select the amount of clusters you would like to see in the graph by typing a whole number between 1 and 20 in the “Cluster” box that is directly below the time input.

You can view the chart generated by the clustering algorithm by clicking on the green “Process” button and then scrolling to the Chart section.
